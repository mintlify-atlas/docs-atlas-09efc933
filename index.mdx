Gentleman Guardian Angel (gga) runs on every `git commit`, sending your staged files to an AI for review against your coding standards. If the code passes, the commit goes through. If it fails, the commit is blocked with a detailed explanation of every violation.

Works with Claude, Gemini, Codex, OpenCode, Ollama, LM Studio, GitHub Models — or any AI CLI you have installed.

## Why GGA?

You have coding standards. Your team ignores them. Code reviews catch issues too late.

GGA runs on every commit — like having a senior developer review every line before it hits the repo.

```
┌─────────────────┐     ┌──────────────┐     ┌─────────────────┐
│   git commit    │ ──▶ │  AI Review   │ ──▶ │  ✅ Pass/Fail   │
│  (staged files) │     │  (any LLM)   │     │  (with details) │
└─────────────────┘     └──────────────┘     └─────────────────┘
```

## Get started in 3 steps

<Steps>
  <Step title="Install gga">
    ```bash
    brew install gentleman-programming/tap/gga
    ```
    Or install manually — see the [Quick Start](/quickstart) guide.
  </Step>
  <Step title="Initialize your project">
    ```bash
    cd ~/your-project
    gga init      # creates .gga config
    gga install   # installs pre-commit hook
    ```
  </Step>
  <Step title="Create your coding standards">
    Create an `AGENTS.md` file with the rules your AI reviewer should enforce. Every commit is now reviewed automatically.

    See [Writing your rules file](/configuration/rules-file) for best practices and examples.
  </Step>
</Steps>

<Note>
  GGA requires only Bash 5.0+ and an AI provider CLI installed on your machine. No API wrappers, no SDKs, no package managers needed.
</Note>
