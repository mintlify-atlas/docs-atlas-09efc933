title: "AI providers"
description: "Setup and configuration reference for every AI provider supported by GGA."
---

GGA works with any AI provider you already have installed. Set `PROVIDER` in your `.gga` config to point to one of the supported options.

## Provider overview

| Provider | Config value | CLI command used | Installation |
|---|---|---|---|
| Claude | `claude` | `echo "prompt" \| claude --print` | [claude.ai/code](https://claude.ai/code) |
| Gemini | `gemini` | `gemini -p "prompt"` | [github.com/google-gemini/gemini-cli](https://github.com/google-gemini/gemini-cli) |
| Codex | `codex` | `codex exec "prompt"` | `npm i -g @openai/codex` |
| OpenCode | `opencode` or `opencode:<model>` | `opencode run "prompt"` | [opencode.ai](https://opencode.ai) |
| Ollama | `ollama:<model>` | `ollama run <model> "prompt"` | [ollama.ai](https://ollama.ai) |
| LM Studio | `lmstudio` or `lmstudio:<model>` | HTTP API call to local server | [lmstudio.ai](https://lmstudio.ai) |
| GitHub Models | `github:<model>` | HTTP API via `gh auth token` | [github.com/marketplace/models](https://github.com/marketplace/models) |

<Note>
  Override the provider for a single run without changing your config file:

  ```bash
  GGA_PROVIDER="gemini" gga run
  ```
</Note>

## Provider setup

<AccordionGroup>
  <Accordion title="Claude">
    Claude is the recommended provider for strict mode and CI/CD pipelines. It reliably follows structured instructions and consistently outputs the `STATUS: PASSED` / `STATUS: FAILED` format.

    **Installation**

    Install the Claude Code CLI from [claude.ai/code](https://claude.ai/code).

    **Verify**

    ```bash
    echo "Say hello" | claude --print
    ```

    **Config**

    ```bash .gga
    PROVIDER="claude"
    ```

    **How GGA calls it**

    ```bash
    printf '%s' "$prompt" | claude --print
    ```
  </Accordion>

  <Accordion title="Gemini">
    Google's Gemini CLI. Built into Antigravity IDE — if you use Antigravity, Gemini is already available in your integrated terminal.

    **Installation**

    Install the Gemini CLI from [github.com/google-gemini/gemini-cli](https://github.com/google-gemini/gemini-cli).

    **Verify**

    ```bash
    echo "Say hello" | gemini
    ```

    You must be authenticated (`gemini login`) before GGA can use it.

    **Config**

    ```bash .gga
    PROVIDER="gemini"
    ```

    **How GGA calls it**

    ```bash
    gemini -p "$prompt"
    ```
  </Accordion>

  <Accordion title="Codex">
    OpenAI's Codex CLI, optimized for code tasks.

    **Installation**

    ```bash
    npm i -g @openai/codex
    ```

    **Verify**

    ```bash
    codex exec "Say hello"
    ```

    **Config**

    ```bash .gga
    PROVIDER="codex"
    ```

    **How GGA calls it**

    ```bash
    codex exec "$prompt"
    ```
  </Accordion>

  <Accordion title="OpenCode">
    OpenCode is a provider-agnostic AI coding CLI. You can use it with its default model or pin a specific model.

    **Installation**

    Install from [opencode.ai](https://opencode.ai).

    **Config**

    <CodeGroup>

    ```bash Default model
    PROVIDER="opencode"
    ```

    ```bash Specific model
    PROVIDER="opencode:anthropic/claude-opus-4-5"
    ```

    </CodeGroup>

    **How GGA calls it**

    ```bash
    # Default model
    opencode run "$prompt"

    # Specific model
    opencode run --model "anthropic/claude-opus-4-5" "$prompt"
    ```
  </Accordion>

  <Accordion title="Ollama (local)">
    Run models locally with Ollama. No API keys required and all data stays on your machine.

    **Installation**

    Install from [ollama.ai](https://ollama.ai) or via Homebrew:

    ```bash
    brew install ollama
    ```

    **Pull a model**

    ```bash
    ollama pull llama3.2
    ollama pull codellama
    ollama pull qwen2.5-coder
    ollama pull deepseek-coder
    ```

    **Config**

    <CodeGroup>

    ```bash Llama 3.2
    PROVIDER="ollama:llama3.2"
    ```

    ```bash CodeLlama
    PROVIDER="ollama:codellama"
    ```

    ```bash Qwen Coder
    PROVIDER="ollama:qwen2.5-coder"
    ```

    ```bash DeepSeek Coder
    PROVIDER="ollama:deepseek-coder"
    ```

    </CodeGroup>

    A model name is required — `PROVIDER="ollama"` without a model name will fail validation.

    **Custom host**

    If your Ollama instance is not running on `localhost:11434`, set `OLLAMA_HOST`:

    ```bash .gga
    OLLAMA_HOST="http://192.168.1.100:11434"
    PROVIDER="ollama:llama3.2"
    ```

    **How GGA calls it**

    GGA prefers the Ollama REST API (via `curl` + `python3`) for clean output. It falls back to the `ollama run` CLI if those tools are not available.

    ```bash
    # API path (preferred)
    curl -s http://localhost:11434/api/generate \
      -d '{"model":"llama3.2","prompt":"...","stream":false}'

    # CLI fallback
    ollama run llama3.2 "$prompt"
    ```

    <Warning>
      Ollama is a pure LLM without file-reading tools. If your `AGENTS.md` uses references to other files (e.g. `- UI guidelines: ui/AGENTS.md`), Ollama cannot read them. Consolidate all rules into a single file when using Ollama.
    </Warning>
  </Accordion>

  <Accordion title="LM Studio (local)">
    LM Studio runs models locally through an OpenAI-compatible HTTP API. No API keys required.

    **Setup**

    <Steps>
      <Step title="Download LM Studio">
        Install from [lmstudio.ai](https://lmstudio.ai).
      </Step>
      <Step title="Download a model">
        Use the model browser inside LM Studio to download a model.
      </Step>
      <Step title="Start the local server">
        Open the **Local Server** tab in LM Studio and start the server. It listens on `http://localhost:1234/v1` by default.
      </Step>
      <Step title="Configure GGA">
        Set `PROVIDER` in your `.gga` file.
      </Step>
    </Steps>

    **Config**

    <CodeGroup>

    ```bash Loaded model (default)
    PROVIDER="lmstudio"
    ```

    ```bash Specific model
    PROVIDER="lmstudio:llama-3.2-3b-instruct"
    ```

    </CodeGroup>

    **Custom host**

    If your LM Studio server is on a different host or port, set `LMSTUDIO_HOST`:

    ```bash .gga
    LMSTUDIO_HOST="http://localhost:8080/v1"
    PROVIDER="lmstudio"
    ```

    **Verify the connection**

    ```bash
    curl http://localhost:1234/v1/models
    ```

    **How GGA calls it**

    GGA calls the OpenAI-compatible `chat/completions` endpoint directly:

    ```bash
    curl -s http://localhost:1234/v1/chat/completions \
      -H "Content-Type: application/json" \
      -d '{"model":"local-model","messages":[{"role":"user","content":"..."}],"stream":false}'
    ```

    `curl` and `python3` must be available on your system. GGA uses `python3` for safe JSON parsing; a sed/grep fallback is used if `python3` is not available.
  </Accordion>

  <Accordion title="GitHub Models">
    Access dozens of hosted models — GPT-4o, GPT-4.1, DeepSeek R1, Grok 3, Phi-4, LLaMA, and more — using your existing GitHub account. No separate API keys needed.

    **Setup**

    <Steps>
      <Step title="Install GitHub CLI">
        ```bash
        brew install gh
        # or see https://cli.github.com
        ```
      </Step>
      <Step title="Authenticate">
        ```bash
        gh auth login
        ```
      </Step>
      <Step title="Configure GGA">
        Set `PROVIDER` with your chosen model.
      </Step>
    </Steps>

    **Config**

    <CodeGroup>

    ```bash GPT-4o
    PROVIDER="github:gpt-4o"
    ```

    ```bash GPT-4.1
    PROVIDER="github:gpt-4.1"
    ```

    ```bash DeepSeek R1
    PROVIDER="github:deepseek-r1"
    ```

    ```bash Grok 3
    PROVIDER="github:grok-3"
    ```

    </CodeGroup>

    A model name is required — `PROVIDER="github"` without a model will fail validation.

    Browse all available models at [github.com/marketplace/models](https://github.com/marketplace/models).

    **How GGA calls it**

    GGA retrieves a token via `gh auth token` and calls the GitHub Models inference endpoint:

    ```bash
    curl -sS https://models.inference.ai.azure.com/chat/completions \
      -H "Authorization: Bearer $(gh auth token)" \
      -H "Content-Type: application/json" \
      -d '{"model":"gpt-4o","messages":[...],"temperature":0.2}'
    ```

    Both `gh` CLI and `curl` must be installed and available in `PATH`.
  </Accordion>
</AccordionGroup>
